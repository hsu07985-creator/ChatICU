## 1. 認證與安全（8 項）

### 1.1 登入流程
- [ ] 開啟 `http://127.0.0.1:3000/login`
- [ ] 輸入 admin / admin → 點擊 Login
- [ ] **預期:** 導向 `/dashboard`，左上角顯示「張管理者 · 系統管理」
- [ ] **驗證 httpOnly cookies:** DevTools → Application → Cookies
  - `chaticu_access`: HttpOnly=true, Path=/
  - `chaticu_refresh`: HttpOnly=true, Path=/auth

### 1.2 帳號鎖定（5 次失敗）
- [ ] 登出後，連續輸入 5 次錯誤密碼（帳號: nurse）
- [ ] **預期:**
  - 第 1-2 次: `帳號或密碼錯誤`
  - 第 3 次: `帳號或密碼錯誤（剩餘 2 次嘗試）`
  - 第 4 次: `帳號或密碼錯誤（剩餘 1 次嘗試）`
  - 第 5 次: `帳號已鎖定，請 15 分鐘後再試`
- [ ] 15 分鐘後用正確密碼登入 → 成功

### 1.3 安全標頭
- [ ] DevTools → Network → 任一 API request → Response Headers
- [ ] **預期 8 項 headers 全部存在:**
  - `X-Frame-Options: DENY`
  - `X-Content-Type-Options: nosniff`
  - `X-XSS-Protection: 1; mode=block`
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Content-Security-Policy: ...`
  - `Cache-Control: no-store`
  - `X-Request-ID: <uuid>`
  - `X-Trace-ID: <uuid>`

### 1.4 登出
- [ ] 點擊側邊欄「登出」按鈕
- [ ] **預期:** 導向 `/login`，cookies 全部清除

### 1.5 Token Refresh
- [ ] 登入後，等待 access token 接近過期（或手動清除 `chaticu_access` cookie）
- [ ] 執行任意 API 操作（如切換頁面）
- [ ] **預期:** 自動刷新 token，不跳轉 login

### 1.6 Session Idle Timeout
- [ ] 登入後靜置超過 30 分鐘（或透過 Redis CLI 手動刪除 `last_activity:*` key）
- [ ] 嘗試操作
- [ ] **預期:** 自動登出，導向 `/login`

### 1.7 速率限制
- [ ] 快速連續發送超過 5 次 login 請求
- [ ] **預期:** 第 6 次返回 `429 Rate limit exceeded`

### 1.8 角色權限 (RBAC)
- [ ] 以 nurse 帳號登入
- [ ] 嘗試存取 `/admin/users`
- [ ] **預期:** 無 admin 側邊欄選項，直接存取返回 403 或空白
