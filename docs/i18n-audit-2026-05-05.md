1. 兩語言對 `count`-style 插值的處理策略不一（zh-TW 沒有用 i18next plural form，en-US 用了 → 但這其實是正確的設計，只是語意成對需要再說明）。
2. 同一個 zh 詞在不同 namespace 翻成不同 en（如「主治醫師」→ Attending / Attending Physician 共存）。
3. 個別 key 名稱與內容不符（例：`renalAbn` 講的是肝功能）。

## Part A：結構問題（高優先 — 但目前不會造成 raw key 顯示）

### A1. zh-TW 缺少 i18next plural-form 對應 key（7 條）

en-US 用了 i18next 的 `_one` / `_other` 後綴提供英文單複數，但 zh-TW 仍用單一 key。i18next runtime 在中文模式下會找 `time.minutesAgo`，找不到 `_one` / `_other` 仍可正常 fallback；但在「英文模式」下若 caller 沒帶 `count`，會 fallback 到 zh-TW 的單一 key（顯示中文）。

**目前實際是否壞掉**：未壞 — 這幾個 key 在程式中都帶了 `count`，i18next 會選對 plural form。但若以後 caller 漏傳 `count`，就會 fallback 到中文。

| Namespace | zh-TW key | en-US key | zh-TW 值 |
|-----------|-----------|-----------|----------|
| common | `time.minutesAgo` | `time.minutesAgo_{one,other}` | `{{count}} 分鐘前` |
| common | `time.hoursAgo` | `time.hoursAgo_{one,other}` | `{{count}} 小時前` |
| common | `time.daysAgo` | `time.daysAgo_{one,other}` | `{{count}} 天前` |
| sidebar | `badge.unreadAria` | `badge.unreadAria_{one,other}` | `未讀通知 {{count}}` |
| sidebar | `badge.newMessages` | `badge.newMessages_{one,other}` | `{{count}} 則新訊息` |
| dashboard | `card.stayDays` | `card.stayDays_{one,other}` | `住院 {{days}} 天` |
| dashboard | `list.columns` | `list.columns_{one,other}` | `{{count}}欄` |

**建議修法**（兩種擇一）：
- (A) **對稱補上 zh-TW 的 `_one` / `_other`**（內容同一個字串，因中文無單複數）— 推薦，最一致。
- (B) **拿掉 en-US 的 `_one` / `_other`，全用單一 key + `(s)`** — 較不自然，不推薦。

### A2. 鍵名「renalAbn」內容卻是肝功能（資料正確性錯誤）

| File | Key | zh-TW | en-US |
|------|-----|-------|-------|
| pharmacy.json | `workstation.assess.advice.renalAbn` | 肝功能異常，建議檢視需肝代謝調整藥物並監測肝功能。 | Liver function abnormal; review hepatic dosing adjustments and monitor LFTs. |

zh 與 en 內容一致，**但 key 名 `renalAbn` 與 `renal`（腎）與 `Abn`（abnormal）的命名暗示「腎功能異常」**，實際內容卻是肝功能。是這條 advice 鍵命名 bug，建議改 key 為 `hepaticAbn`（連同 callsite 一起改）。

### A3. 翻譯落差但不算結構問題

`patient-detail.header.stayDays`：

- zh：`住院 {{days}} 天`（描述已住的累積天數）
- en：`Day {{days}} of stay`（描述「住院第 N 天」）

兩種寫法都常見，但語意微差：「Day 5 of stay」是序數第五天；「住院 5 天」可能是「累計 5 天」。同一份資料下不會有歧義，但保留與 dashboard `card.stayDays_*` 一致（都用 `Day N of stay`）即可。

## Part D：跨 namespace 術語不一致

### D1.「入院診斷」→ Admission Dx vs Admission Diagnosis

| 出現位置 | en-US | 場景 |
|----------|-------|------|
| `dashboard:card.diagnosisLabel` | Admission Diagnosis | 病患卡片正面（空間夠） |

**建議**：表格欄頭可保留 `Admission Dx`（節省欄寬），其他全部改 `Admission Diagnosis`。需在 glossary 加註「列表縮寫例外」。

### D2.「主治醫師」→ Attending vs Attending Physician

| 出現位置 | en-US |
|----------|-------|
| `dashboard:edit.labels.physician` | Attending Physician |
| `patients:list.table.physician` | **Attending**（縮寫） |
| `patients:list.filters.physicianGroup` | Attending Physician |
| `patients:create.labels.physician` | Attending Physician |

| `patients:discharged.physicianLabel` | Attending Physician |
| `patients:discharged.table.physician` | **Attending** |
| `patient-tabs:summary.infoCard.rows.physician` | **Attending** |

**建議**：統一為 `Attending Physician`（除非欄寬不夠，則明顯標 `Attending`，並在 glossary 註記）。`patient-tabs.summary.infoCard` 不是表格而是詳情卡，**應改為 Attending Physician**。

### D3.「藥物管理」/「藥物統計」→ namespace 與 sidebar 對齊

| 出現位置 | zh-TW | en-US |
|----------|-------|-------|

### D4.「藥事評估」→ Pharmacy Review vs Pharmacy Assessment

| 出現位置 | zh-TW | en-US |
|----------|-------|-------|
| `sidebar:groups.pharmacyAssessment` | 藥事評估 | Pharmacy Review |
| `pharmacy:workstation.assess.reportView.title` | 藥事評估報告 | Pharmacy Assessment Report |

**建議**：統一為 `Pharmacy Assessment`（更貼近「評估」原意；`Review` 偏 retrospective，與 ICU 即時審查不太搭）。一旦選定，sidebar 也跟著改。

### D5. 單位縮寫（year / day）

| zh-TW | en-US 縮寫 | 出現位置 |
|-------|-----------|----------|
| 歲 | y/o | `dashboard:card.ageYears`、`patient-detail:header.ageGender`、`patients:list.ageSuffix`、`patient-tabs:summary.ageSuffix` |
| 歲 | **y** | `pharmacy:workstation.patientSelect.labels.ageSuffix` |
| 歲 | y/o | `pharmacy:workstation.assess.reportView.ageWithUnit` |
| 天 | d | `patients:list.icuDaysSuffix`、`patients:list.ventilatorDaysSuffix`、`patient-detail:header.stayDays` 用 `Day {{days}} of stay` |
| 天 | days | `pharmacy:dosage` etc. |

**建議**：`y/o` 統一；`d` 在表格欄頭用，敘述句用 `days`。`pharmacy.workstation.patientSelect.labels.ageSuffix = "y"` 改為 `"y/o"`。

### D6.「插管」→ Intubated vs Airway

| 出現位置 | zh-TW | en-US |
|----------|-------|-------|
| `dashboard:metrics.intubated` | 插管人數 | Intubated |
| `patients:list.table.intubation` | 插管 | **Airway**（合理 — 該欄同時顯示「氣切」） |
| `patients:list.notIntubated` | 未插管 | **No** |
| `patient-detail:header.intubated` | 插管中 | Intubated |
| `patient-tabs:summary.flags.intubated` | 插管中 | Intubated |

**建議**：`patients:list.notIntubated` = `No` 太隱晦，且該欄頭叫 `Airway`，因此 `No` 完全失去語意。建議改為 `None`（airway support = none）或 `—`。

### D7.「重複用藥」→ Duplicate Therapy vs Duplicate Medication vs Duplicates

| 出現位置 | zh-TW | en-US |
|----------|-------|-------|
| `sidebar:items.duplicates` | 重複用藥 | Duplicate Therapy |
| `medications:duplicateBadges.summaryLabel` | 重複用藥 | Duplicate Therapy |
| `medications:riskCard.duplicateLabel` | 重複用藥 | **Duplicate**（單字 — 在 risk card 縮寫合理） |
| `medications:tab.main.viewDuplicate` | 重複用藥 ({{count}}) | Duplicate ({{count}}) |
| `dashboard:duplicateBadge.ariaLabel` | 重複用藥警示 | **Duplicate medication alert** |
| `pharmacy:workstation.assess.panel.duplicate` | 重複用藥 | **Duplicates** |

**建議**：標題 / sidebar 用 `Duplicate Therapy`（醫療正式詞）；卡片標籤可縮為 `Duplicates` 或 `Duplicate`；aria-label 用完整 `Duplicate therapy alert`。需在 glossary 規範。

### D8.「介入紀錄」→ Intervention Record / History Record

| 出現位置 | zh-TW | en-US |
|----------|-------|-------|
| `pharmacy:adviceStats.tabAdvice` | 藥事介入紀錄 | Intervention Records |
| `pharmacy:adviceStats.editRecord` | 編輯歷史紀錄 | Edit history record |
| `pharmacy:adviceStats.recordUpdated` | 歷史紀錄已更新 | History record updated |
| `score-trend:history.title` | 歷史紀錄 | History |

中文一律「歷史紀錄」/「介入紀錄」，en 在 advice 頁混用 `intervention record` 與 `history record` 指同一份資料。**建議**：統一為 `Intervention record`，因為這就是該功能的本體（藥師介入），其餘地方該是「介入紀錄的歷史」 → `Intervention history`。

### D9.「處理中」→ Processing（一致 — OK）

`auth:changePassword.submitting` 用 `Processing...`（含 ellipsis），其他多為 `Processing`（無 ellipsis）。建議 toast / button label 一致用 `Processing...`（與 zh 原文 `處理中...` 對齊）。

### D10.「載入中」→ Loading（一致）

`pharmacy:library.proposals.loading` 與 `library.detail.page.loading` 是 `Loading`（無 ellipsis）；其他是 `Loading...`。建議：含 `...` 表進行中，純 `Loading` 表狀態 — 但目前 `library.proposals.loading` 也是 button submit-pending 場景，建議加 `...` 一致。

## Appendix：實作建議

### 補 plural-form 範例（A1）

zh-TW/common.json：
```json
"time": {
  "justNow": "剛剛",
  "minutesAgo_one": "{{count}} 分鐘前",
  "minutesAgo_other": "{{count}} 分鐘前",
  "hoursAgo_one": "{{count}} 小時前",
  "hoursAgo_other": "{{count}} 小時前",
  "daysAgo_one": "{{count}} 天前",
  "daysAgo_other": "{{count}} 天前"
}
```

中文兩個 plural form 內容相同，但結構對稱可確保 i18next 不 fallback。callsite 仍寫 `t('time.minutesAgo', { count: 5 })`，i18next 自動選 form。
