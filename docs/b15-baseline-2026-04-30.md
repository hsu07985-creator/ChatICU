## 4. Baseline 結果

### 4.1 TIMING（server-side `[CHAT][TIMING]`，n=18）

| 指標 | p50 | p95 | p99 | min | max |
|---|---|---|---|---|---|
| **`ttft_ms`** | **1235** | **1931** | 2637 | 654 | 2637 |
| `snapshot_ms` | 0 | **6030** | 6053 | 0 | 6053 |
| `pre_llm_ms` | 561 | 584 | 590 | 559 | 590 |

| `total_ms`（server 收到 → 第一 token）| 2413 | 8350 | 8361 | 1636 | 8361 |
| `sys_prompt_chars` | 1838 | 2185 | 2185 | 1838 | 2185 |

### 4.2 CACHE（`[CHAT][CACHE]`，n=18）

| 指標 | p50 | p95 | p99 | min | max |
|---|---|---|---|---|---|
| `prompt_tokens` | 1810 | 2539 | 3153 | 1237 | 3153 |
| `cached_tokens` | 1280 | 1792 | 2304 | **0** | 2304 |
| **`hit_ratio_pct`** | **70%** | 87% | 95% | **0%** | 95% |
| `completion_tokens` | 588 | 1040 | 1077 | 78 | 1077 |

### 4.3 First-turn vs subsequent-turn 差異

樣本結構：6 sessions × 3 turn = 18 call。**6 個 first-turn + 12 個 subsequent-turn**。

從 `snapshot_ms` 分布反推：
- **First turn**：`snapshot_ms` ~6000ms（建 snapshot）、`hit_ratio` 0%（首次無 cache）
- **Subsequent turn**：`snapshot_ms` 0ms（reuse）、`hit_ratio` 70-95%

→ snapshot 建立的 6s 集中在 first turn；subsequent turn TTFT 約 1.2s 並有高 cache hit。

## 5. Path Status 更新（baseline 後）

### Path A — Prompt cache ✅ **驗證為 work**

| 證據 | 結果 |
|---|---|
| `cache_hit_ratio_p50` | **70%**（> 50% threshold）|
| `prompt_tokens` 全 ≥ 1024 token threshold | ✅ min=1237 通過 OpenAI cache 門檻 |
| `cache_hit_ratio` 在 subsequent turn 高（推估 70-95%）、first turn 0% | ✅ 行為符合預期 |
| `sys_prompt_chars` 跨 turn 在同 session 應 byte-stable | 推測符合，未深入 byte-by-byte verify |

→ **不需要 debug Path A prefix stability**。可直接進 Path B。

### Path B — Snapshot compression：**主要槓桿在這**

從 baseline 看出兩個 snapshot 相關痛點：

1. **First-turn `snapshot_ms` p95 = 6030ms**：建 snapshot 本身要 6 秒（DB queries 到 Supabase Sydney 串行 RTT 累積）
2. **subsequent-turn TTFT p50 = ~1.2s**：但 prompt 仍含完整 snapshot text → 即使 70% cache hit，剩 30% non-cached 部分仍要過 LLM

Path B 同時解這兩個：
- 壓縮 snapshot → 減少 prompt size → LLM TTFT 同步降
- 如果 snapshot 結構性簡化，DB query 數也可能減少 → first-turn snapshot_ms 降

**風險**：snapshot 是 LLM 唯一 patient context，壓太多會讓 LLM 答錯。**必須**先寫 eval test cases。
